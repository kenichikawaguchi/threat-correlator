# 直近30日間のハニーポット観測レポート ― 26件検知（Critical13件・High3件・Unknown10件）

## はじめに
ハニーポット「AIBOT RADAR」は、攻撃者が実際に試行するマルウェア・エクスプロイトを大量に捕捉できる環境です。本レポートは、2026年5月末から2026年6月末までの30日間にハニーポットで観測された **26件の CVE**（Critical 13、High 3、危険度不明 10）を分析し、日本のセキュリティエンジニア・システム管理者が即座に取るべき対策を示します。観測は「ET Signature」ベースで自動検出し、各脆弱性の初回検知日時とシグネチャ数も併記しています。

---

## 🔴 Critical（CVSS 9.0 以上） 13件

### SAP SE – SAP NetWeaver Visual Composer development server  
**CVE-2025-31324**（CVSS 10.0）  

- **概要**：Visual Composer のメタデータアップローダが認証なしで実行でき、攻撃者は任意の実行可能バイナリをサーバーにアップロード可能。  
- **攻撃シナリオ**：攻撃者は HTTP POST でメタデータファイルを送信し、サーバー上でコードが実行され、機密情報の取得やサービス停止が可能になる。  
- **推奨対策**：  
  1. 公式パッチ（SAP Note 3594142）を即時適用。  
  2. 未使用の Visual Composer コンポーネントは無効化、外部からの直接アクセスを防ぐファイアウォール規則を追加。  

### Ivanti – Endpoint Manager Mobile  
**CVE-2023-35078**（CVSS 10.0）  

- **概要**：認証バイパスにより、未認証のリクエストで管理 API にアクセス可能。  
- **攻撃シナリオ**：攻撃者はモバイル端末管理サーバーに対し認証情報なしでデバイス情報や設定変更を取得／実行できる。  
- **推奨対策**：  
  1. Ivanti が提供する 2023‑12‑01 以降のパッチを適用。  
  2. 管理 API の IP 制限と、TLS 終端の強化を実施。  

### ConnectWise – ScreenConnect  
**CVE-2024-1709**（CVSS 10.0）  

- **概要**：代替パスを利用した認証バイパスにより、遠隔から管理コンソールへ直接アクセス可能。  
- **攻撃シナリオ**：攻撃者は特定の URL パラメータを細工し、認証済みユーザーとして画面共有やリモート操作を取得。  
- **推奨対策**：  
  1. バージョン 23.9.7 以前を使用中なら、最新 23.10.2 へアップデート。  
  2. 管理画面へのインターネット公開を停止し、VPN 経由のみ許可。  

### Roundcube Webmail  
**CVE-2025-49113**（CVSS 9.9）  

- **概要**：`_from` パラメータの検証不備により、PHP オブジェクトデシリアライズが可能。認証ユーザーが遠隔コード実行できる。  
- **攻撃シナリオ**：認証済みユーザーが細工した URL をメール本文に埋め込み、他ユーザーがクリックするとサーバー側でコードが実行される。  
- **推奨対策**：  
  1. 1.5.10 以降、もしくは 1.6.11 以降にアップグレード。  
  2. Web アプリケーションファイアウォール（WAF）で PHP オブジェクトデシリアライズパターンをブロック。  

### Cisco – HyperFlex HX Data Platform  
**CVE-2021-1497**（CVSS 9.8）  

- **概要**：管理 UI のコマンドインジェクションが可能で、未認証のリモート攻撃者が任意コマンドを実行できる。  
- **攻撃シナリオ**：攻撃者は特定の API エンドポイントに細工したパラメータを送信し、システム内部でシェルコマンドが起動。  
- **推奨対策**：  
  1. Cisco が提供する最新ソフトウェアリリースへ更新（2022‑04‑01 以降）。  
  2. 管理 UI へのアクセスは社内 IP に限定し、強制的に多要素認証（MFA）を導入。

### LiteSpeed – LiteSpeed Cache プラグイン  
**CVE-2024-44000**（CVSS 9.8）  

- **概要**：認証情報が不適切に保護され、認証バイパスが可能。  
- **攻撃シナリオ**：攻撃者はプラグインの管理画面に直接アクセスし、管理者アカウントを取得。  
- **推奨対策**：  
  1. 6.5.0.1 以降に更新。  
  2. プラグインディレクトリへの直接アクセスを `.htaccess` で制限。  

### WP Automatic  
**CVE-2024-27954**（CVSS 9.3）  

- **概要**：パストラバーサルと SSRF により任意ファイルの取得や内部サーバーへのリクエストが可能。  
- **攻撃シナリオ**：攻撃者は `/wp-content/plugins/automatic/` 配下のファイルパスを操作し、サーバー上の機密情報をダウンロード。  
- **推奨対策**：  
  1. バージョン 3.92.0 以上へ更新。  
  2. `open_basedir` と `allow_url_fopen` を無効化し、外部リクエストを制限。  

### Fortinet – FortiOS / FortiProxy  
**CVE-2023-27997**（CVSS 9.2）  

- **概要**：ヒープバッファオーバーフローにより、リモートから任意コード実行が可能。  
- **攻撃シナリオ**：特定の SSL‑VPN リクエストを細工し、ファイアウォール上でコードが実行され、内部ネットワークが完全取得される。  
- **推奨対策**：  
  1. 公式パッチ（FG‑IR‑23‑097）を適用。  
  2. 不要な SSL‑VPN 機能は無効化し、管理アクセスは IP 制限で保護。  

### Progress – MOVEit Transfer  
**CVE-2024-5806**（CVSS 9.1）  

- **概要**：SFTP モジュールの認証バイパス。未認証でも SFTP 接続が確立でき、ファイル操作が可能。  
- **攻撃シナリオ**：攻撃者は SFTP クライアントで認証情報なしに接続し、機密ファイルの取得・改ざんを実行。  
- **推奨対策**：  
  1. 2023.0.11 以降、2023.1.6、2024.0.2 以上へアップデート。  
  2. SFTP アクセスは内部ネットワークに限定し、接続ログを監視。  

### Fortinet – FortiClient EMS  
**CVE-2026-21643**（CVSS 9.1）  

- **概要**：SQL インジェクションにより未認証で任意コード実行が可能。  
- **攻撃シナリオ**：特 crafted HTTP リクエストでデータベースクエリを改ざんし、管理者権限のコマンドが実行される。  
- **推奨対策**：  
  1. 7.4.4 以上へ更新（公式パッチ適用）。  
  2. Web アプリケーションファイアウォールで `SELECT` 系クエリの検査を強化。  

### Microsoft – Exchange Server  
**CVE-2021-34473**（CVSS 9.1）  

- **概要**：ProxyShell と呼ばれるリモートコード実行脆弱性。認証なしで Exchange のバックエンドにコードを注入できる。  
- **攻撃シナリオ**：攻撃者は特定の HTTP ヘッダーと文字列を組み合わせ、Exchange の内部サービスを乗っ取り、メールボックス情報や内部ネットワークへの踏み台に利用。  
- **推奨対策**：  
  1. Microsoft が提供する 2021‑12‑01 以降の累積的アップデートを適用。  
  2. Exchange の外部公開を最小限に抑え、必要なら Azure AD の条件付きアクセスポリシーを導入。

## まとめと対策の優先順位

1. **Critical 脆弱性の即時パッチ適用**  
   - 特に **認証バイパス系**（CVE‑2025‑31324、CVE‑2024‑1709、CVE‑2023‑42793/27198、CVE‑2024‑44000） は、認証なしでシステム全体が制御されるリスクが最も高い。  
   - すべての対象製品でベンダーが提供する **最新マイナーバージョン** もしくは **セキュリティパッチ** を本日中に適用。

2. **外部公開サービスの縮小**  
   - 管理コンソールや API エンドポイントがインターネットに露出しているケース（ConnectWise ScreenConnect、ColdFusion、FortiOS VPN、Jenkins CLI 等）は、**VPN・IP制限** でアクセスを内部に限定。

3. **認証強化と多要素認証 (MFA) の導入**  
   - Exchange Server、Cisco HyperFlex、Ivanti EPMM など、認証バイパスが報告された製品は必ず **MFA** を有効化し、不要なデフォルトアカウントは無効化。

4. **Web アプリケーションファイアウォール (WAF) のルール追加**  
   - PHP オブジェクトデシリアライズ、XSS、パストラバーサル、オープンリダイレクトといった典型的な攻撃パターンを **シグネチャベース** でブロック。

5. **監視とログ保全**
