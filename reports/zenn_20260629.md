# 2026年5月30日～2026年6月28日 観測 CVE 26件（Critical 13件、High 3件、Unknown 10件）

## はじめに
AIBOT RADAR が収集した *実際に日本向けハニーポットで観測された* 脆弱性情報をもとに、直近30日間に確認された 26 件の CVE をまとめました。ハニーポットは外部からの不正アクセス試行を受動的に捕捉し、試行回数・発信国・観測期間を自動で記録します。本稿では **Critical（CVSS ≥ 9.0）** を中心に、攻撃シナリオと即時取るべき対策を解説します。

## 🔴 Critical（CVSS 9.0 以上）観測 CVE

### SAP SE SAP NetWeaver (Visual Composer development server)
- **観測情報**：2026年5月29日に初めて観測され、6月29日時点で計 **92回**、オランダ・米国からプローブされている。  
- **概要**：Visual Composer のメタデータアップローダが認証なしで実行可能。攻撃者は任意のバイナリをサーバへアップロードでき、OS の権限でコード実行が可能になる。  
- **攻撃シナリオ**：  
  1. `/visualcomposer/metadataUploader` へ HTTP POST でマルウェアバイナリを送信。  
  2. サーバ側でバイナリが実行され、システム全体が乗っ取られる（機密情報漏洩・サービス停止）。  
- **推奨対策**：  
  - 対象コンポーネントを **2025 年 12 月のパッチ**（SAP Note 3594142）へ更新。  
  - アップロード先ディレクトリへの **ファイル実行権限を除外**。  
  - ネットワークレベルで **該当エンドポイントへのアクセス制御**（IP ACL）を実装。

### Ivanti Endpoint Manager Mobile
- **観測情報**：2026年6月1日に初めて観測、計 **1回**、オランダからのプローブ。  
- **概要**：認証バイパスにより、未認証ユーザーが管理機能へ直接アクセス可能。  
- **攻撃シナリオ**：認証不要の API エンドポイント `/api/v1/devices` を叩き、デバイス情報取得や設定変更を実行。  
- **推奨対策**：  
  - **2023 年 12 月版** 以上にアップデート（Ivanti のセキュリティパッチ適用）。  
  - API 認証を **必須** にし、外部からの直接呼び出しをファイアウォールでブロック。

---

### ConnectWise ScreenConnect
- **観測情報**：2026年6月1日に初めて観測、計 **1回**、オランダ。  
- **概要**：代替経路を利用した認証バイパスにより、リモートデスクトップ管理コンソールへ不正アクセス可能。  
- **攻撃シナリオ**：特定の URL パラメータを操作し、セッション認証を迂回し管理画面にログイン。  
- **推奨対策**：  
  - **23.9.8 以降** の最新リリースへアップデート。  
  - 管理画面への **VPN 限定** アクセスと強力な多要素認証（MFA）を導入。

### Roundcube Webmail (version < 1.5.10)
- **観測情報**：2026年6月17日に初めて観測、計 **1回**、英国。  
- **概要**：`_from` パラメータの検証不足に起因する PHP オブジェクトデシリアライズで、認証済ユーザーが任意コード実行。  
- **攻撃シナリオ**：メール作成時に不正 `_from` パラメータを付与し、サーバ側でシリアライズオブジェクトが復元され任意 PHP コードが実行される。  
- **推奨対策**：  
  - **1.5.10 以上** へバージョンアップ（公式パッチ適用）。  
  - `settings/upload.php` の入力サニタイズを **WAF で補強**。

### Cisco HyperFlex HX Data Platform
- **観測情報**：2026年6月1日に初めて観測、計 **1回**、オランダ。  
- **概要**：Web 管理 UI に複数のコマンドインジェクション脆弱性があり、未認証リモートからシステムコマンド実行が可能。  
- **攻撃シナリオ**：管理画面の検索フィールドに `; rm -rf /` などのシェルコマンドを注入し、実行させる。  
- **推奨対策**：  
  - Cisco の **Security Advisory** に基づく **最新ファームウェア** への更新。  
  - 管理 UI への **IP 制限** と **MFA** 強制。

### JetBrains TeamCity (version < 2023.05.4)
- **観測情報**：2026年6月1日に初めて観測、計 **1回**、オランダ。  
- **概要**：認証バイパスにより未認証ユーザーが管理者権限でコード実行可能。  
- **攻撃シナリオ**：`/admin` エンドポイントに特定のクエリ文字列を付与し、管理コンソールへ不正アクセス。  
- **推奨対策**：  
  - **2023.05.4 以降** のリリースへアップデート。  
  - 管理画面は **内部ネットワークのみからアクセス** 可能にし、MFA を適用。

### JetBrains TeamCity (version < 2023.11.4)
- **観測情報**：2026年6月1日に初めて観測、**5回**、オランダ。  
- **概要**：認証バイパスで管理者操作が可能になる脆弱性。  
- **攻撃シナリオ**：同様に `/admin` へ特定リクエストを送信し、ビルド設定変更や機密情報取得を実行。  
- **推奨対策**：  
  - **2023.11.4 以降** にアップデート。  
  - 管理インターフェースの **IP ホワイトリスト** と **MFA**。

### LiteSpeed Technologies LiteSpeed Cache (version < 6.5.0.1)
- **観測情報**：2026年6月1日に初めて観測、計 **1回**、オランダ。  
- **概要**：認証情報が不適切に保護され、未認証で管理者アカウント取得が可能。  
- **攻撃シナリオ**：`/wp-admin/admin-ajax.php` 経由でキャッシュプラグインの認証チェックをバイパスし、管理者権限で設定変更。  
- **推奨対策**：  
  - **6.5.0.1 以上** へのアップデート。  
  - WordPress 本体の **強力なパスワード** と **二要素認証** を導入。

### WP Automatic (Automatic) (version < 3.92.0)
- **観測情報**：2026年6月1日に初めて観測、計 **1回**、オランダ。  
- **概要**：パストラバーサルと SSRF により任意ファイル取得や内部リクエストが可能。  
- **攻撃シナリオ**：`/wp-content/plugins/automatic/download.php?file=../../wp-config.php` で設定ファイル取得、さらに内部サービスへリクエスト送信。  
- **推奨対策**：  
  - **3.92.0 以上** へ更新。  
  - プラグインディレクトリへの **実行権限除去**、Web アプリケーションファイアウォールでパストラバーサルをブロック。

### Fortinet FortiOS / FortiProxy (複数バージョン)
- **観測情報**：2026年6月1日に初めて観測、**2回**、オランダ。  
- **概要**：ヒープバッファオーバーフロー（CWE‑122）により、特定リクエストで任意コード実行が可能。  
- **攻撃シナリオ**：SSL‑VPN の認証ハンドシェイク中に細工したパケットを送信し、ファイアウォール内部でコードが実行される。  
- **推奨対策**：  
  - FortiOS‑7.2.5 以降、FortiProxy‑7.2.4 以降へ **アップデート**。  
  - SSL‑VPN は **IP 制限** と **MFA** を必須化。

---

### Progress MOVEit Transfer (SFTP モジュール)
- **観測情報**：2026年6月1日に初めて観測、**1回**、オランダ。  
- **概要**：認証バイパスにより SFTP 接続が無認証で成立。  
- **攻撃シナリオ**：`/sftp` エンドポイントへ認証情報なしで接続し、ファイルのアップロード・ダウンロードが可能になる。  
- **推奨対策**：  
  - **2023.0.11** 以降、**2024.0.2** 以降へ更新。  
  - SFTP は **IP ACL** と **クライアント証明書認証** を併用。

### Fortinet FortiClientEMS 7.4.4
- **観測情報**：2026年6月1日に初めて観測、**516回**、オーストラリア・ベルギー・ブラジル・チリ・フランス・インド・インドネシア・日本・オランダ・シンガポール・韓国・スウェーデン・台湾・英国・米国から継続的にプローブ。観測回数が **100回超** であり、**大規模なスキャン** が実施されていることが判明。  
- **概要**：SQL インジェクション（CWE‑89）により、未認証の HTTP リクエストで任意 SQL を実行可能。  
- **攻撃シナリオ**：`/api/v1/endpoint?filter=string' OR '1'='1` のようなクエリでデータベースを抽出、さらに管理者権限でコマンド実行。  
- **推奨対策**：  
  - **7.4.5** 以降の **公式パッチ** を即適用。  
  - Web アプリケーションファイアウォールで **SQLi シグネチャ** を有効化。  
  - 管理 API は **IP 制限** と **MFA** を必須。

### Microsoft Exchange Server (複数バージョン・Cumulative Update)
- **観測情報**：2026年6月2日に初めて観測、**13回**、米国からのプローブ。  
- **概要**：ProxyShell と呼ばれるリモートコード実行脆弱性（CVE‑2021‑34473）。認証不要で Exchange の内部 API を利用し、任意コードを実行できる。  
- **攻撃シナリオ**：`/ecp` エンドポイントに特 crafted URL を送り、バックエンドで PowerShell スクリプトが実行され、メールボックス情報取得やスパム送信が可能になる。  
- **推奨対策**：  
  - Microsoft が提供する **2021‑34473 用パッチ**（KB5000871 など）を全 Exchange サーバに適用。  
  - ECP への外部アクセスは **VPN 経由のみ** に限定。  
  - 監査ログの **詳細レベル** を有効化し、異常な `Authorization` ヘッダーを検知。

## まとめと対策の優先順位

以下のチェックリストは **緊急度が高い順に★の数で示しています**。★が多いほど、速やかに実施すべきです。  

- [ ] ★★★★★ **Fortinet FortiClientEMS (CVE‑2026‑21643)** – 516回観測の大規模スキャン。公式パッチ適用と WAF で SQLi 防御。  
- [ ] ★★★★★ **SAP NetWeaver Visual Composer (CVE‑2025‑31324)** – 92回観測。アップロード機能のパッチ適用とネットワーク制限。  
- [ ] ★★★★★ **Microsoft Exchange Server (CVE‑2021‑34473)** – 13回観測。全サーバにマイクロソフト提供パッチ適用、外部からの ECP アクセス遮断。  
- [ ] ★★★★★ **Microsoft MOVEit Transfer (CVE‑2024‑5806)** – 1回観測だが重要インフラで広く使用。SFTP 認証バイパスパッチ適用。  
- [ ] ★★★★☆ **Grafana (CVE‑2025‑4123)** – 268回観測。XSS／SSRF 防止の CSP 設定、匿名アクセス無効化。  
- [ ] ★★★★☆ **JetBrains TeamCity (CVE‑2023‑42793 / CVE‑2024‑27198)** – 認証バイパス。最新リリースへの更新と管理 UI の IP 制限。  
- [ ] ★★★★☆ **LiteSpeed Cache (CVE‑2024‑44000)** – 認証情報漏洩。プラグイン更新と WordPress の強固な認証導入。  
- [ ] ★★★★☆ **Fortinet FortiOS / FortiProxy (CVE‑2023‑27997)** – 2回観測。ファームウェア更新と SSL‑VPN の二要素認証。  
- [ ] ★★★☆☆ **Adobe ColdFusion (CVE‑2024‑20767)** – 3回観測。最新版更新と管理画面の外部公開停止。  
- [ ] ★★★☆☆ **その他 High CVE** – Grafana 以外はそれぞれパッチ適用とアクセス制御の見直し。  

### 最後に
ハニーポットが捕捉した情報は「**実際に日本でも狙われている**」という強い示唆です。特に **CVE‑2026‑21643** のように観測回数が数百回に上るケースは、攻撃者が自動化ツールで大規模にスキャンしている可能性が高く、放置すれば被害が拡大します。  
本記事で紹介した対策を **優先度順に実施** し、脆弱性情報は常に最新のベンダーアドバイザリと照らし合わせて管理してください。安全なシステム運用は「**パッチ適用 + 防御層の硬化**」の組み合わせが最も効果的です。
