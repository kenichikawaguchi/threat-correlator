# 【AIBOT RADAR】日本向けハニーポットで観測された Critical CVE 13件と攻撃トレンド ― 2026年6月版

## はじめに
AIBOT RADAR が収集した日本向けハニーポット（AIBOT RADAR）において、直近 30 日間で **26 件** の CVE が観測されました。そのうち **Critical（CVSS 9.0 以上）** が 13 件、**High（CVSS 7.0〜8.9）** が 3 件、**CVSS 未確認** が 10 件です。本記事では、特に危険度の高い Critical CVE を中心に、観測結果と対策をまとめます。

## 🔴 Critical（CVSS 9.0 以上）の観測 CVE  

### CVE-2025-31324 – SAP NetWeaver Visual Composer 開発サーバ  
**観測結果**  
- 初回観測日: 2026-05-29  
- 最終観測日: 2026-06-29  
- 観測回数: 92 回  
- 観測国: Netherlands, United States  

**脆弱性概要**  
Visual Composer のメタデータアップローダが認証を要求せず、任意の実行可能バイナリをサーバにアップロードできる状態です。これにより、攻撃者はホスト上でコードを実行し、機密情報の窃取やサービス停止といった被害が発生する可能性があります。  

**攻撃例（想定）**  
攻撃者は特別に細工したバイナリをアップロードし、Web アプリケーション層から OS コマンドを実行。結果として、バックドア設置や内部ネットワークへの横展開が行われることが想定されます。  

**推奨対策**  
- 該当する Visual Composer コンポーネントを **2025‑12‑01 以降のパッチ** に更新  
- アップロード機能を外部から遮断するファイアウォールルールの適用  
- 不要な開発サーバはインターネットから完全に切り離す

### CVE-2023-35078 – Ivanti Endpoint Manager Mobile  
**観測結果**  
- 初回観測日: 2026-06-01  
- 最終観測日: 2026-06-01  
- 観測回数: 1 回  
- 観測国: Netherlands  

**脆弱性概要**  
認証バイパスにより、正規ユーザーでない者が管理機能や機密リソースへ直接アクセスできる状態です。  

**攻撃例（想定）**  
攻撃者は認証不要の API エンドポイントを呼び出し、デバイス情報や設定を取得・変更できる可能性があります。  

**推奨対策**  
- Ivanti が提供する **2023‑12‑15 以降のアップデート** を適用  
- API アクセスに IP 制限や VPN 経由のみの利用を実装  
- ログ監視で不審な API 呼び出しを検知

### CVE-2025-49113 – Roundcube Webmail (< 1.5.10)  
**観測結果**  
- 初回観測日: 2026-06-17  
- 最終観測日: 2026-06-17  
- 観測回数: 1 回  
- 観測国: United Kingdom  

**脆弱性概要**  
`_from` パラメータが検証されず、PHP オブジェクトデシリアライズが実行できるため、認証ユーザーが任意コードを実行できる状態です。  

**攻撃例（想定）**  
攻撃者は細工した URL を送付し、ターゲットがクリックすると PHP オブジェクトがデシリアライズされ、サーバ上でコマンドが実行される可能性があります。  

**推奨対策**  
- Roundcube を **1.5.10 以上** にアップグレード  
- Web アプリケーションファイアウォールで危険なパラメータをブロック  
- 不要なメールサーバは外部からの直接アクセスを遮断

### CVE-2021-1497 – Cisco HyperFlex HX Data Platform  
**観測結果**

- 最終観測日: 2026-06-01  
- 観測回数: 1 回  
- 観測国: Netherlands  

**脆弱性概要**  
Web 管理インターフェースに複数のコマンドインジェクション脆弱性があり、認証されていないリモート攻撃者が任意コマンドを実行できる可能性があります。  

**攻撃例（想定）**

**推奨対策**  
- Cisco が提供する **2021‑12‑10 以降のパッチ** を適用  
- 管理インターフェースは内部ネットワークのみで利用し、外部からの直接アクセスを遮断  
- コマンドインジェクション防止のため、入力値のサニタイズを徹底

### CVE-2024-27954 – WP Automatic (< 3.92.0)  
**観測結果**  
- 初回観測日: 2026-06-01  
- 最終観測日: 2026-06-01  
- 観測回数: 1 回  
- 観測国: Netherlands  

**脆弱性概要**  
ディレクトリトラバーサルと SSRF が可能で、任意ファイルの取得や内部リソースへのリクエストが実行できる状態です。  

**攻撃例（想定）**  
攻撃者は `/wp-content/plugins/wp-automatic/...` にパスを細工し、サーバ上の任意ファイルを取得、さらに内部サービスへリクエストを送ることで情報漏洩や内部ネットワークの踏み台化が想定されます。  

**推奨対策**  
- プラグインを **3.92.0 以上** に更新  
- `open_basedir` や `disable_functions` でファイル操作を制限  
- 不要なプラグインは即削除

### CVE-2023-27997 – Fortinet FortiOS / FortiProxy (複数バージョン)  
**観測結果**  
- 初回観測日: 2026-06-01  
- 最終観測日: 2026-06-01  
- 観測回数: 2 回  
- 観測国: Netherlands  

**脆弱性概要**  
ヒープベースのバッファオーバーフローにより、特定のリクエストで任意コード実行が可能になる欠陥です。  

**攻撃例（想定）**  
攻撃者は細工した SSL-VPN パケットを送信し、メモリ上のバッファを上書きしてシェルコードを実行。結果として、VPN ゲートウェイが乗っ取られる可能性があります。  

**推奨対策**  
- FortiOS 7.2.5 以降、FortiProxy 7.2.4 以降にアップデート  
- SSL-VPN の外部公開は最小限に抑え、IP アクセス制限を適用  
- IDS/IPS で該当パケットパターンを検知
